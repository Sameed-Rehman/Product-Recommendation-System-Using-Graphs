# Product Recommendation System Using Graphs

This project builds a graph-based product recommendation system using Python, RFM scoring, and product co-occurrence in orders.

It was developed collaboratively and demonstrates how transactional order data can be transformed into weighted product recommendations.

## Project Idea
The system treats each product as a node in a directed graph.

- If two products appear in the same order, an edge is created between them.
- Edge weight is influenced by the target product's RFM-based importance score.
- Recommendations for a product are generated by sorting outgoing neighbors by edge weight.

## Features
- Generates sample transactional order data.
- Computes RFM metrics (Recency, Frequency, Monetary) per product.
- Converts RFM metrics to quantile-based scores.
- Builds an RFM-weighted directed graph using NetworkX.
- Visualizes:
  - RFM summary table (in `main.py`)
  - Product recommendation graph
- Supports interactive CLI recommendations (`main.py`).

## Repository Structure
- `main.py`: End-to-end pipeline with RFM table visualization, graph visualization, and interactive recommendation prompt.
- `working.py`: Script variant focused on graph construction and graph detail output.

## How It Works
1. Generate synthetic orders with product, quantity, price, and date.
2. Compute product-level RFM values:
	- Recency: Days since latest purchase
	- Frequency: Number of occurrences
	- Monetary: Quantity x Price aggregated per product
3. Convert each RFM metric into quantile scores.
4. Compute a combined product weight:

	`Weight = 0.4 * R_Score + 0.3 * F_Score + 0.3 * M_Score`

5. Build directed edges from co-purchased products.
6. Rank outgoing edges as recommendations for the selected product.

## Installation
1. Clone the repository.
2. Install dependencies:

```bash
pip install pandas numpy networkx matplotlib
```

## Run
Use either script:

```bash
python main.py
```

or

```bash
python working.py
```

For `main.py`, enter a product code (A-Z) in the terminal to view recommendations.

## Example Recommendation Logic
If product `A` co-occurs often with high-weight products `C`, `F`, and `M`, the system will recommend those products in descending weighted order.

## Notes and Limitations
- Current dataset is synthetic and small (`num_orders = 5`).
- RFM scoring uses quantiles; very small datasets can make quantile bins sensitive.
- This is a foundational prototype and can be extended with real data, evaluation metrics, and model validation.

## Future Improvements
- Use real transactional datasets.
